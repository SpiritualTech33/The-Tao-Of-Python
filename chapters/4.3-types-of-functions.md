# 4.3 Types of Functions

## Not All Functions Are Equal

Functions can be classified by their relationship with state, side effects, and what they do. Understanding these types improves your design decisions.

---

## Pure Functions — The Gold Standard

A pure function always produces the same output for the same input, and has **no side effects.**

```python
# Pure — same input always → same output, no external impact
def add(a: float, b: float) -> float:
    return a + b

def calculate_batting_average(hits: int, at_bats: int) -> float:
    return hits / at_bats if at_bats > 0 else 0.0
```

Pure functions are:
- **Easy to test** — no setup needed, just call with inputs, check output
- **Easy to reason about** — no hidden state or dependencies
- **Safe to call anywhere** — no risk of unexpected side effects

Prefer pure functions whenever possible.

---

## Impure Functions — With Side Effects

An impure function interacts with the outside world: printing, writing files, modifying state, making network requests.

```python
# Impure — produces a side effect (prints to screen)
def log_result(result: float) -> None:
    print(f"[RESULT] {result:.3f}")

# Impure — modifies external state
total_hits = 0   # External state

def record_hit() -> None:
    global total_hits
    total_hits += 1   # Modifies external variable
```

Side effects are often necessary (you have to print, write files, call APIs). The goal is to **isolate** them — keep the logic pure and the side effects at the edges of your program.

---

## Lambda Functions — Anonymous and Small

A lambda is a small, anonymous function defined in a single expression:

```python
# Full function
def square(x):
    return x ** 2

# Lambda equivalent
square = lambda x: x ** 2

# Common use — as an argument to another function
players = [{"name": "Cosmos", "avg": 0.342}, {"name": "Miguel", "avg": 0.298}]

sorted_by_avg = sorted(players, key=lambda p: p["avg"], reverse=True)
# Sorted highest average first

numbers = [1, 2, 3, 4, 5, 6]
evens = list(filter(lambda n: n % 2 == 0, numbers))   # [2, 4, 6]
doubled = list(map(lambda n: n * 2, numbers))          # [2, 4, 6, 8, 10]
```

Use lambdas for short, throwaway functions. For anything complex, use a proper `def`.

---

## Higher-Order Functions

Functions that accept or return other functions:

```python
# Accepts a function as argument
def apply(func, value):
    return func(value)

apply(str.upper, "cosmos")    # "COSMOS"
apply(len, "cosmos")          # 6

# Returns a function — closure
def make_multiplier(factor: int):
    def multiply(x: float) -> float:
        return x * factor
    return multiply

double = make_multiplier(2)
triple = make_multiplier(3)

print(double(5))   # 10
print(triple(5))   # 15
```

---

## Recursive Functions

A function that calls itself:

```python
def factorial(n: int) -> int:
    """Compute n! = n × (n-1) × ... × 1"""
    if n <= 1:        # Base case — stops the recursion
        return 1
    return n * factorial(n - 1)   # Recursive case

factorial(5)   # 5 × 4 × 3 × 2 × 1 = 120
```

Every recursive function needs:
1. A **base case** that stops the recursion
2. A **recursive case** that moves toward the base case

Use recursion when the problem is naturally recursive (trees, fractals, mathematical sequences). Use loops for simple iteration.

---

## Void Functions

Functions that produce actions but return nothing:

```python
def print_player_card(player: dict) -> None:
    """Display a player's stats card."""
    print(f"{'='*30}")
    print(f"  {player['name'].upper()}")
    print(f"  Position: {player['position']}")
    print(f"  AVG: {player['avg']:.3f}")
    print(f"{'='*30}")
```

The `-> None` annotation makes the intent explicit: this function is called for its effects, not its return value.
