# 5.3 LBYL and EAFP — Two Philosophies of Caution

## Before You Step

There are two ways a careful person crosses a street.

The first looks both ways, sees no cars, then steps. *Look Before You Leap.*

The second steps, and knows how to respond if a car appears. *Easier to Ask Forgiveness than Permission.*

Python has names for both of these approaches.

---

## LBYL — Look Before You Leap

**LBYL** checks conditions *before* taking an action. If the condition is not met, don't attempt the action:

```python
# LBYL — check first, then act
def safe_divide_lbyl(a: float, b: float) -> float:
    if b != 0:
        return a / b
    return 0.0

# LBYL — check if key exists before accessing
def get_player_average_lbyl(player: dict) -> float:
    if "average" in player:
        return player["average"]
    return 0.0

# LBYL — check if file exists before opening
import os

def read_config_lbyl(filepath: str) -> str:
    if os.path.exists(filepath):
        with open(filepath) as f:
            return f.read()
    return ""
```

LBYL is explicit. You can read it and immediately understand what conditions are required.

---

## EAFP — Easier to Ask Forgiveness than Permission

**EAFP** attempts the action directly and handles the exception if it occurs:

```python
# EAFP — try it, catch the failure
def safe_divide_eafp(a: float, b: float) -> float:
    try:
        return a / b
    except ZeroDivisionError:
        return 0.0

# EAFP — access directly, handle missing key
def get_player_average_eafp(player: dict) -> float:
    try:
        return player["average"]
    except KeyError:
        return 0.0

# EAFP — open the file, handle if missing
def read_config_eafp(filepath: str) -> str:
    try:
        with open(filepath) as f:
            return f.read()
    except FileNotFoundError:
        return ""
```

EAFP is optimistic. It assumes the common case succeeds and handles the rare failure.

---

## Which One Is "Right"?

Python's culture — and the Zen of Python — leans toward **EAFP**. The language was designed this way. Exceptions are cheap in Python. The `try` block has almost no overhead when no exception occurs.

But the real answer is: **use the one that makes your intent clearest.**

**LBYL is better when:**
- The check is simple and cheap
- You want to validate input at a function's entry point
- The failure condition is as likely as the success condition

**EAFP is better when:**
- The success case is much more likely than the failure
- The check and the action cannot be separated safely (race conditions)
- You need to handle multiple failure types

---

## The Race Condition Problem

LBYL has a subtle danger: between the check and the action, the world can change.

```python
# LBYL — but vulnerable to race conditions
if os.path.exists(filepath):
    # Another process deletes the file here
    with open(filepath) as f:   # FileNotFoundError anyway!
        return f.read()

# EAFP — handles the reality of what happens, not just what checked
try:
    with open(filepath) as f:
        return f.read()
except FileNotFoundError:
    return ""
```

In concurrent programs, LBYL's check can become stale. EAFP handles reality as it is, not as it was a moment ago.

---

## The Pythonic Pattern

In practice, Python code often combines both. Validate at the boundary with LBYL (reject bad input early), and use EAFP inside the logic (trust the flow, catch what breaks):

```python
def process_player_stats(player: dict) -> float:
    # LBYL — validate at the boundary
    if not isinstance(player, dict):
        raise TypeError(f"Expected dict, got {type(player).__name__}")

    # EAFP — trust the structure, handle missing fields
    try:
        hits = player["hits"]
        at_bats = player["at_bats"]
        return hits / at_bats
    except KeyError as e:
        raise ValueError(f"Player data missing required field: {e}") from e
    except ZeroDivisionError:
        return 0.0
```

---

## The Philosophical Reading

LBYL is the philosophy of the guard. It stands at the door and asks for credentials before letting anyone in.

EAFP is the philosophy of the dojo. It opens its doors, and it knows how to respond to whatever enters.

Neither is fearful. Both are prepared. The difference is *where the preparation lives* — in the conditions you check, or in the responses you have ready.

Choose deliberately.
