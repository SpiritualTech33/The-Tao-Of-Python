# 3.7 Advanced Flow Control

## break — Exit the Loop Now

`break` immediately exits the innermost loop, regardless of the remaining iterations:

```python
# Find first even number
numbers = [1, 3, 5, 4, 7, 8]
for n in numbers:
    if n % 2 == 0:
        print(f"Found: {n}")
        break   # Stop — no need to continue
# Found: 4

# Search with break
def find_player(roster: list, target: str) -> int:
    for i, player in enumerate(roster):
        if player == target:
            return i   # return also exits the loop
    return -1
```

## continue — Skip This Iteration

`continue` skips the rest of the current iteration and jumps to the next:

```python
# Process only active players
for player in players:
    if not player["active"]:
        continue   # Skip inactive players
    process(player)

# Skip even numbers
for n in range(10):
    if n % 2 == 0:
        continue
    print(n)   # 1, 3, 5, 7, 9
```

## pass — Do Nothing

`pass` is a no-op — syntactically valid, does nothing. Used as a placeholder:

```python
# During development — implement later
def calculate_obp():
    pass   # TODO: implement

# Empty except block (usually bad practice — but valid)
try:
    risky_operation()
except SomethingMinor:
    pass   # Intentionally ignored

# Empty class
class FutureFeature:
    pass
```

## else on Loops

Python has an unusual but powerful feature: `else` on loops. The `else` block runs **only if the loop completed without a `break`:**

```python
# Search for an item
roster = ["Cosmos", "Miguel", "Luis"]
target = "Ana"

for player in roster:
    if player == target:
        print(f"Found: {target}")
        break
else:
    print(f"{target} not in roster")   # Runs because no break occurred
# "Ana not in roster"

# Common use: searching + reporting result
def find_and_report(data: list, value) -> bool:
    for item in data:
        if item == value:
            break
    else:
        return False   # Completed without finding
    return True
```

The `else` on loops is one of Python's most misunderstood features. Once understood, it becomes a clean alternative to flag variables.

## Comprehensions as Loops

List, dictionary, and set comprehensions are compact loops:

```python
# List comprehension
squares = [x**2 for x in range(10)]

# With condition
evens = [x for x in range(20) if x % 2 == 0]

# Dictionary comprehension
lengths = {word: len(word) for word in ["tao", "zen", "void"]}

# Set comprehension
unique_lengths = {len(word) for word in ["tao", "zen", "void", "cosmos"]}

# Generator expression — like list comprehension but lazy (no list created)
total = sum(x**2 for x in range(1000))   # Computes without building the list
```
