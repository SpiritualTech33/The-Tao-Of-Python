# 1.5 Specific Types — Going Deeper

## The Nature of Numbers

### Integers

Integers in Python have **no size limit.** Unlike most languages where an int overflows at 2³² or 2⁶⁴, Python integers grow as large as memory allows.

```python
# Underscores for readability (Python 3.6+)
population = 8_000_000_000
national_debt = 33_000_000_000_000

# Integer operations
print(17 // 5)   # 3  → integer (floor) division
print(17 %  5)   # 2  → remainder (modulo)
print(2 ** 10)   # 1024 → exponentiation
```

The modulo operator `%` is more powerful than it looks. It tells you the *remainder after division* — which means it tells you about cycles, patterns, even/odd, and much more.

### Floats

Floats represent decimal numbers using IEEE 754 floating-point standard. This brings precision limits:

```python
print(0.1 + 0.2)   # 0.30000000000000004  ← not exactly 0.3!
```

This is not Python being wrong. It is a fundamental property of how computers represent decimal numbers in binary. When precision matters, use `round()` or the `decimal` module:

```python
result = round(0.1 + 0.2, 2)   # 0.3
```

## Strings in Depth

Strings are **immutable sequences of Unicode characters.** Two important words:

**Immutable** — once created, a string cannot be modified. Every string operation returns a *new* string:
```python
original = "python"
upper = original.upper()   # New string — original unchanged
print(original)            # "python"
print(upper)               # "PYTHON"
```

**Unicode** — Python 3 handles all human languages natively. Emojis, Chinese characters, Arabic script — all valid:
```python
greeting = "你好 🐍 مرحبا"   # Perfectly valid Python
```

### Multi-line Strings

```python
haiku = """
Old pond —
a frog jumps in,
sound of water.
"""

sql_query = """
    SELECT name, batting_avg
    FROM players
    WHERE active = TRUE
"""
```

### Raw Strings

When you need backslashes literally (file paths, regex):
```python
# Without raw string — backslash is escape character
path = "C:\\Users\\cosmos\\Documents"

# With raw string — backslash is literal
path = r"C:\Users\cosmos\Documents"
```

## Booleans — The Logic at the Core

Booleans are actually **integers** in Python. `True == 1` and `False == 0`. This is not a quirk — it is intentional design:

```python
print(True + True)    # 2
print(True * 5)       # 5
print(sum([True, False, True, True]))  # 3 — count of True values
```

**Truthy and Falsy** — Python evaluates more than just `True/False` in boolean context:

```python
# Falsy values — evaluate as False
bool(0)        # False
bool(0.0)      # False
bool("")       # False
bool([])       # False
bool({})       # False
bool(None)     # False

# Everything else is truthy
bool(1)        # True
bool(-1)       # True
bool("zen")    # True
bool([0])      # True — list with one element
```

This means you can write:
```python
name = "Cosmos"
if name:       # True if not empty
    print(f"Hello, {name}")

items = []
if not items:  # True if empty
    print("Nothing to process")
```

## None — The Void

`None` is Python's representation of the void — the absence of a value. It is not zero, not empty string, not `False`. It is *nothing.*

```python
result = None
print(result is None)    # True — use 'is', not '=='
print(result == None)    # Also True, but 'is' is preferred
```

Use `None` as a default value when you don't know yet what something should be, or to signal "this function found nothing."
