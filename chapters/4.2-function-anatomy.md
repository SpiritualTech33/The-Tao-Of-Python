# 4.2 The Anatomy of Functions

## Dissecting the Form

```python
def calculate_batting_average(hits: int, at_bats: int = 0) -> float:
    """
    Calculate batting average as hits divided by at_bats.
    Returns 0.0 if at_bats is zero.
    """
    if at_bats == 0:
        return 0.0
    return hits / at_bats
```

Every part is intentional:

---

## `def` — The Declaration

`def` is short for *define.* It says: "I am creating a named, reusable unit of code."

The keyword comes first — it is a contract between you and Python: *"I am declaring a function here."*

---

## The Name — A Promise

`calculate_batting_average` is not just a label. It is a promise to every future reader: *"Call this, and it will calculate a batting average."*

Rules for naming:
- Lowercase with underscores: `snake_case`
- Use verbs or verb phrases: `calculate_`, `get_`, `validate_`, `create_`, `is_`
- Be specific: `calculate_batting_average` > `calc_avg` > `do_thing`
- Length is fine if it adds clarity

---

## Parameters and Arguments

**Parameters** are the names in the function definition.
**Arguments** are the values passed when calling.

```python
def add(a, b):      # a, b are PARAMETERS
    return a + b

add(3, 5)           # 3, 5 are ARGUMENTS
```

**Default values** make parameters optional:

```python
def greet(name: str, greeting: str = "Hello") -> str:
    return f"{greeting}, {name}!"

greet("Cosmos")              # "Hello, Cosmos!"
greet("Cosmos", "Namaste")   # "Namaste, Cosmos!"
```

Rules for defaults:
- Parameters with defaults must come *after* parameters without
- Default values are evaluated once at definition time — use `None` as default for mutable objects

```python
# Bug — mutable default value
def add_item(item, items=[]):    # Same list reused across calls!
    items.append(item)
    return items

# Correct
def add_item(item, items=None):
    if items is None:
        items = []
    items.append(item)
    return items
```

---

## Type Hints

Type hints are optional but recommended:

```python
def calculate_average(hits: int, at_bats: int) -> float:
    ...

def process_name(name: str | None = None) -> str:
    ...

from typing import list   # For older Python
def get_team(size: int) -> list[str]:
    ...
```

Type hints do not enforce types at runtime — Python remains dynamically typed. But they serve as documentation, enable IDE autocomplete, and allow static analysis tools to catch errors.

---

## The `return` Statement

A function can return any value, or return `None` implicitly:

```python
def add(a, b):
    return a + b        # Returns the sum

def log_event(msg):
    print(f"[LOG] {msg}")
    # Implicitly returns None

# Multiple return values — actually a tuple
def get_stats(hits, at_bats):
    avg = hits / at_bats if at_bats else 0.0
    obp = avg * 1.1
    return avg, obp     # Returns tuple (0.34, 0.374)

average, on_base = get_stats(34, 100)   # Unpack immediately
```

---

## Calling Functions

```python
# Positional arguments — order matters
calculate_batting_average(34, 100)

# Keyword arguments — order doesn't matter
calculate_batting_average(at_bats=100, hits=34)

# Mix — positional first, then keyword
calculate_batting_average(34, at_bats=100)
```

Using keyword arguments when calling improves readability, especially for functions with many parameters.
