# 1.7 Operators and Reserved Words

## Operators — The Language of Transformation

Operators are the verbs of Python. They transform, compare, combine.

### Arithmetic

```python
a, b = 10, 3

a + b      # 13   — addition
a - b      # 7    — subtraction
a * b      # 30   — multiplication
a / b      # 3.333 — division (always float)
a // b     # 3    — floor division (integer result)
a % b      # 1    — modulo (remainder)
a ** b     # 1000 — exponentiation
```

The modulo operator `%` is more powerful than it looks. It reveals cycles and patterns:
```python
n % 2 == 0    # is n even?
n % 7 == 0    # does n fall on a weekly cycle?
n % 360       # normalize an angle
```

### Comparison — Return True or False

```python
5 == 5     # True  — equal
5 != 3     # True  — not equal
5 > 3      # True  — greater than
5 < 3      # False — less than
5 >= 5     # True  — greater or equal
5 <= 4     # False — less or equal
```

### Logical

```python
True and False    # False — both must be True
True or False     # True  — at least one must be True
not True          # False — inversion
```

**Short-circuit evaluation:** Python stops evaluating as soon as the result is certain:
```python
# If first is False, second is never checked
False and expensive_function()   # → False (function not called)

# If first is True, second is never checked
True or expensive_function()     # → True (function not called)
```

### Assignment Operators

```python
x = 10
x += 3    # x = x + 3 → 13
x -= 2    # x = x - 2 → 11
x *= 2    # x = x * 2 → 22
x //= 3   # x = x // 3 → 7
x **= 2   # x = x ** 2 → 49
```

### Identity and Membership

```python
# Identity — same object in memory
x = None
x is None       # True  — use 'is' for None
x is not None   # False

# Membership — contained in sequence
"z" in "zen"                  # True
"x" in ["a", "b", "c"]        # False
"key" in {"key": "value"}     # True
```

## Reserved Words — The Sacred 35

These belong to the language. You cannot use them as variable names:

```
False    None     True     and      as       assert
async    await    break    class    continue def
del      elif     else     except   finally  for
from     global   if       import   in       is
lambda   nonlocal not      or       pass     raise
return   try      while    with     yield
```

Each one carries a specific meaning in the language's grammar. Attempting to use them as variable names raises a `SyntaxError` immediately. Learn to see them as the syntax **has its own vocabulary** — and these words belong to it, not to you.

## Operator Precedence

When multiple operators appear in one expression, Python follows mathematical precedence:

```
()           ← parentheses first
**           ← exponentiation
+x, -x, ~x  ← unary operators
*, /, //, %  ← multiplicative
+, -         ← additive
comparisons  ← ==, !=, <, >, <=, >=
not          ← logical not
and          ← logical and
or           ← logical or
```

When in doubt, **use parentheses.** They make intent explicit and eliminate ambiguity:

```python
# Ambiguous
result = 2 + 3 * 4 ** 2

# Clear
result = 2 + (3 * (4 ** 2))   # 50
```
