# 2.4 Sets

## The Philosophy of Uniqueness

A set answers one fundamental question: **among all these elements, which ones are truly distinct?**

It is the mathematical concept of a set — a collection where each element exists exactly once, with no duplicates and no guaranteed order.

```python
# Duplicates are eliminated automatically
unique = {3, 1, 4, 1, 5, 9, 2, 6, 5, 3}
print(unique)   # {1, 2, 3, 4, 5, 6, 9}  — order not guaranteed

# From a list — a common deduplication pattern
names = ["Cosmos", "Miguel", "Cosmos", "Luis", "Miguel"]
unique_names = set(names)
print(unique_names)   # {"Cosmos", "Miguel", "Luis"}
```

## Creating Sets

```python
# With curly braces
planets = {"Mercury", "Venus", "Earth", "Mars"}

# From any iterable
vowels = set("aeiouaeiou")   # {"a", "e", "i", "o", "u"}
unique_numbers = set([1, 2, 2, 3, 3, 3])   # {1, 2, 3}

# Empty set — MUST use set(), not {}
empty = set()       # Correct
not_empty = {}      # This creates an empty DICTIONARY
```

## Set Operations — Where Sets Shine

Sets implement the full vocabulary of mathematical set theory:

```python
team_a = {"Cosmos", "Miguel", "Luis"}
team_b = {"Luis", "Carlos", "Ana"}

# Union — all elements from both
team_a | team_b        # {"Cosmos", "Miguel", "Luis", "Carlos", "Ana"}
team_a.union(team_b)   # Same

# Intersection — only elements in both
team_a & team_b             # {"Luis"}
team_a.intersection(team_b) # Same

# Difference — in a but not in b
team_a - team_b            # {"Cosmos", "Miguel"}
team_a.difference(team_b)  # Same

# Symmetric difference — in one but not both
team_a ^ team_b                          # {"Cosmos", "Miguel", "Carlos", "Ana"}
team_a.symmetric_difference(team_b)      # Same
```

## Membership Testing — O(1)

The most important performance advantage of sets: **checking membership is nearly instant**, regardless of size.

```python
# With a list — must scan every element O(n)
allowed_list = ["admin", "editor", "viewer"]
"admin" in allowed_list   # Scans list

# With a set — hash lookup O(1)
allowed_set = {"admin", "editor", "viewer"}
"admin" in allowed_set    # Instant, always
```

For large collections where membership tests are frequent, convert to a set. The difference in speed can be enormous.

## Modifying Sets

```python
tags = {"python", "zen", "code"}

tags.add("tao")           # Add one element
tags.update(["spirit", "mind"])  # Add multiple
tags.remove("zen")        # Remove — raises KeyError if missing
tags.discard("missing")   # Remove — no error if missing
popped = tags.pop()       # Remove and return arbitrary element
tags.clear()              # Remove all
```

---

## When to Use Sets

- Remove duplicates from a list
- Test membership in a large collection
- Find common elements between two groups
- Find elements unique to one group

Sets are **not** for ordered data, indexed access, or storing duplicates. For those, use lists.
