## The Dual Role

In all previous chapters, the tests were given to you. You were the builder — the tests were the criteria.

Here, you are building the criteria too. You are deciding:

- Which cases are worth testing?
- What are the edge cases?
- What should failure look like?

This is the hardest skill. Anyone can implement a function that someone else specified. The engineer specifies *and* implements.

---

## Kata 2 — The Stack

A Stack is a Last-In, First-Out data structure. Like a stack of plates: you add to the top, you take from the top.

Your job is to implement `Stack` and use the provided fixtures (`empty_stack`, `populated_stack`) to test it.

Study the fixtures before you write the class. The fixtures tell you what the class needs to do.

---

## Kata 6 — SimpleLogger

A logger records messages. `SimpleLogger` should:
- Accept messages via a `log(message)` method
- Store messages in order
- Return all messages via `get_logs()` → a list of strings
- Support clearing via `clear()`

Your fixture should provide a fresh logger with a few pre-logged messages, so tests don't need to set up from scratch.
