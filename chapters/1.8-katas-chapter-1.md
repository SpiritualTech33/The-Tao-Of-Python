## What Are Katas?

In martial arts, a **kata** is a choreographed sequence of movements practiced until it becomes instinct — until the body executes without the mind having to command each step.

Code katas work the same way. They are small, focused exercises you practice until the patterns become second nature. Not to memorize syntax — to **build instinct.**

---

## The 7 Katas

Open the file `katas/chapter_01_katas.py`. Inside you will find 7 functions — each one returns `pass` (nothing). Your mission: implement each function so that all tests pass.

| Kata | Concept |
|------|---------|
| 1. `describe_variable` | Types and f-strings |
| 2. `build_greeting` | String formatting |
| 3. `clean_and_title` | String methods |
| 4. `get_initials` | Slicing and split |
| 5. `compute` | Operators and exceptions |
| 6. `count_words` | String processing |
| 7. `is_palindrome` | Logic and slicing |

## How to Run

From your terminal, navigate to the `katas/` folder:

```bash
cd path/to/Python-From-The-Void/katas
```

Run all Chapter 1 katas:
```bash
python -m pytest chapter_01_katas.py -v
```

Run a specific kata:
```bash
python -m pytest chapter_01_katas.py::test_describe_variable -v
```

---

## Reading the Output

**When a test fails:**
```
FAILED chapter_01_katas.py::test_build_greeting[cosmos-25-...]
AssertionError: assert None == "Hello, Cosmos. You are 25 years old."
```

`None` means your function returned nothing — you haven't implemented it yet. Replace `pass` with your logic.

**When a test passes:**
```
PASSED chapter_01_katas.py::test_build_greeting[cosmos-25-...]
```

**When all pass:**
```
7 passed in 0.08s
```

That is the sound of understanding.

## The Protocol

1. Open `chapter_01_katas.py`
2. Read the docstring of each function carefully
3. Implement the function — **do not modify the tests**
4. Run pytest, observe what fails
5. Fix. Run again. Repeat.
6. When all 7 pass — breathe. Move to Chapter 2.

*One kata at a time. One line at a time. One understanding at a time.*
