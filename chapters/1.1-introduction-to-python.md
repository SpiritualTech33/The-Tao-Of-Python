# 1.1 Introduction to Python

## Why Does Python Exist?

Before *how*, ask *why.*

Guido van Rossum was a Dutch programmer at the Centrum Wiskunde & Informatica (CWI) in the Netherlands. He was frustrated: existing languages were either too powerful and complex (C, C++) or too limited. There was no middle ground between *can do anything* and *easy enough to think clearly.*

So he built one. Python was born from a single belief:

> *"Code is read far more often than it is written."*

This one idea shaped everything. Python prioritizes **the human reader over the machine.** That is radical. Most languages do the opposite.

---

## Guido's Patterns of Thought

Understanding how Guido thought explains every design decision in Python:

**Pragmatism over purity.** Python is not purely object-oriented or purely functional. It takes what works. *"Does it solve real problems? Then it's worth it, even if it's not theoretically perfect."*

**Empathy with the reader.** He always thought: *"Will a programmer reading this in 6 months understand it easily?"* That is pure empathy — thinking about the human on the other side.

**Gradual perfectionism.** Python started simple and evolved. Not perfect from day one — iterated, listened, improved. Python 2 → Python 3 was painful, but necessary.

**Long-term vision.** He didn't optimize for trends. He designed for durability. That's why Python remains dominant 35 years later.

---

## What Python Is

**Python is a translator between humans and machines.**

Computers understand only zeros and ones. Humans think in words, logic, abstract concepts. Python is the bridge between both worlds.

Key characteristics:

- **Interpreted** — reads and executes line by line. No compilation step. Write → run → see.
- **High-level** — abstracts hardware complexity. You focus on *what*, not *how the CPU manages RAM.*
- **Dynamically typed** — no need to declare types. Python infers them.
- **General purpose** — web, data science, AI/ML, automation, science, finance. Python does everything.

## The Zen of Python

Type this anywhere Python runs:

```python

```

You'll see 19 aphorisms. The essential ones:

> *Beautiful is better than ugly.*
> *Explicit is better than implicit.*
> *Simple is better than complex.*
> *Readability counts.*

This is not decoration. It is the contract. The Zen of Python is not a checklist — it is a **mentality you embody through practice.** Like breathing in meditation. You don't think about it. You live it.

## Python and AI — A Symbiotic Relationship

Python *is* the language of AI. Why?

- Researchers are mathematicians and scientists, not systems programmers. Python lets them express complex algorithms without fighting syntax.
- The entire ecosystem lives in Python: TensorFlow, PyTorch, NumPy, Pandas, scikit-learn.
- Rapid prototyping: test an idea in minutes, not days.

Tools like GitHub Copilot show what happens when Python and AI combine: **a loop where AI builds the tools that build better AI.**

You are entering this loop. Remember that.
