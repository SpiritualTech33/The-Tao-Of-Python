# 2.1 Collections in Python

## What Is a Collection?

A collection is a structure that groups **multiple values under a single name.**

But think deeper. A collection is not just "many data together." It is a **form of organizing multiplicity** — making the many behave as one coherent unit.

```python
# Without collections — fragmented
player1_name = "Cosmos"
player2_name = "Miguel"
player3_name = "Luis"
# ...player27_name = ?

# With a collection — unified
team = ["Cosmos", "Miguel", "Luis"]
```

The second version thinks in concepts, not in variables.

---

## Why Collections Exist

The world does not present itself in isolation. You naturally think in:
- A **list** of tasks
- A **set** of players
- An **inventory** of items
- A **profile** with name, age, position

Collections exist so programs resemble how we actually think.

---

## Python's Four Core Collections

| Collection | Ordered | Mutable | Unique | Key-Value |
|------------|---------|---------|--------|-----------|
| **List**   | ✅ | ✅ | ❌ | ❌ |
| **Tuple**  | ✅ | ❌ | ❌ | ❌ |
| **Set**    | ❌ | ✅ | ✅ | ❌ |
| **Dict**   | ✅* | ✅ | Keys only | ✅ |

*Dictionaries preserve insertion order since Python 3.7.

---

## The Four Philosophical Profiles

**List** — the river. Ordered, flowing, mutable. You can add, remove, rearrange. Good for sequences that change over time.

**Tuple** — the stone in the riverbed. Ordered, fixed, immutable. Good for data that should not change — coordinates, configurations, records.

**Set** — the night sky. Unordered, unique. Each star exists once. Good for membership testing and eliminating duplicates.

**Dictionary** — the oracle. Named. Every concept has a meaning accessible by its name. Good for anything that needs lookup by key.

## Nested Collections

Collections can contain other collections — this is how complex real-world data is represented:

```python
# A list of dictionaries — the most common real-world pattern
team = [
    {"name": "Cosmos", "position": "pitcher",   "avg": 0.342},
    {"name": "Miguel", "position": "catcher",   "avg": 0.298},
    {"name": "Luis",   "position": "shortstop", "avg": 0.315},
]

# Access: list index → dictionary key
print(team[0]["name"])   # "Cosmos"
print(team[1]["avg"])    # 0.298

# List all player names
names = [player["name"] for player in team]
# ["Cosmos", "Miguel", "Luis"]
```

This pattern — a list of dictionaries — appears in virtually every real Python program. Know it well.
