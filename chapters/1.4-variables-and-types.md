# 1.4 Variables and Types

## Why Variables Exist

Before syntax, the *why.*

A variable is how you give something a **name** and a **home in memory.**

Without variables, every computation would be use-once and forgotten. Variables are how programs *remember.* They are the nouns of your code — the things that exist and can be referred to, transformed, and passed around.

When you write:
```python
age = 25
```

You are doing three things simultaneously:
1. Allocating memory for a value
2. Storing the value `25` in it
3. Binding the name `age` to that location

The name is what matters most. Names should be so clear that comments become unnecessary.

## Python's Fundamental Types

Python has five primitive types. Know them not as technical categories but as **kinds of reality:**

**`int`** — whole numbers. Discrete. Countable.
```python
at_bats = 100
strikeouts = 12
year = 2025
```

**`float`** — decimal numbers. Continuous. Measurable.
```python
batting_average = 0.342
height_meters = 1.83
pi = 3.14159
```

**`str`** — text. Language. The interface between code and human consciousness.
```python
name = "Cosmos De La Cruz"
position = 'pitcher'
mantra = "I am the algorithm"
```

**`bool`** — truth values. The two states of logical reality.
```python
is_active = True
has_error = False
```

**`NoneType`** — the absence of value. Not zero. Not empty string. *Nothing.*
```python
result = None     # Value not yet assigned
config = None     # Will be set later
```

---

## Dynamic Typing

Python infers types automatically — you never declare them:

```python
x = 42        # int
x = 3.14      # now float
x = "cosmos"  # now str
```

Check the type of anything with `type()`:
```python
print(type(42))        # <class 'int'>
print(type(3.14))      # <class 'float'>
print(type("hello"))   # <class 'str'>
print(type(True))      # <class 'bool'>
print(type(None))      # <class 'NoneType'>
```

---

## Type Conversion

Convert between types explicitly:
```python
# str → int
age_str = "25"
age_int = int(age_str)        # 25

# int → float
price = float(42)             # 42.0

# number → str
display = str(0.342)          # "0.342"

# anything → bool
bool(0)       # False
bool(1)       # True
bool("")      # False
bool("zen")   # True
bool(None)    # False
```

## Naming Variables Well

This is where most beginners fail — and where masters separate themselves.

```python
# Bad — meaningless
x = 25
y = 100
z = x / y

# Good — self-documenting
hits = 25
at_bats = 100
batting_average = hits / at_bats
```

**Rules:**
- Lowercase with underscores: `batting_average`, not `BattingAverage`
- Descriptive: the name should tell you what the variable *is*
- No single letters except for loop counters (`i`, `j`) or mathematical formulas
- Constants in UPPERCASE: `MAX_ATTEMPTS = 3`, `TAX_RATE = 0.15`

The name is a promise. Keep it.
