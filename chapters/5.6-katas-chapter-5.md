## Kata 3 — The Custom Exception

This kata introduces `InvalidAverageError` — a custom exception you must define and raise. A batting average must live between 0.0 and 1.0. Any value outside that range is not just wrong — it is *meaningless.*

Your function does not guess what to return. It speaks:

```
InvalidAverageError: Batting average must be between 0.0 and 1.0, got 1.5
```

*Implement `InvalidAverageError` at the top of the file before implementing the function.*

## Kata 5 — retry

This is the most architectural kata. You are implementing a function that takes another function and tries calling it up to `max_attempts` times. If it succeeds, return the result. If it fails every time, raise the last exception.

This pattern appears in real production systems: network calls, flaky APIs, file locks.

Think carefully about how you capture and re-raise the exception from the final attempt.
