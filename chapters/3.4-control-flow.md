# 3.4 Control Flow — Teaching Code to Think

## Why Control Flow Exists

Until now, your code has been a river flowing in one direction — line by line, no choices, no repetition. Inevitable.

But consider: a program that can only do one thing, in one order, regardless of input, is not a program. It is a recipe. **A script.**

For code to be genuinely useful, it must be able to:
- Make **decisions** based on conditions
- **Repeat** actions until something is satisfied
- **Redirect** its path based on what it encounters

Control flow is how you give code the ability to **reason about reality** and respond accordingly.

---

## The Three Structures

All control flow in every programming language reduces to three fundamental structures:

**Sequence** — default. One statement after another.

**Selection** — branching based on condition.
```
if (condition) → take this path
else           → take that path
```

**Iteration** — repeating based on condition.
```
while (condition) → keep doing this
for each item    → do this for each
```

These three, combined in any depth and order, can express any computable algorithm. This is not a Python fact — it is a mathematical theorem (the Structured Program Theorem, 1966). Three structures. All computation.

---

## The Boolean Heart

All control flow depends on **boolean expressions** — expressions that evaluate to `True` or `False`.

```python
# Simple boolean expressions
age >= 18               # True or False
name == "Cosmos"        # True or False
"key" in dictionary     # True or False
value is None           # True or False

# Compound expressions
age >= 18 and has_id    # True only if BOTH are True
is_member or has_ticket # True if EITHER is True
not is_expired          # Inverts the boolean
```

**Short-circuit evaluation** — Python stops as soon as the result is determined:
```python
# If first condition is False, second is never evaluated
has_permission and perform_action()

# If first condition is True, second is never evaluated
from_cache or fetch_from_server()
```

This is not just optimization — it is a programming pattern. You will use it constantly.

---

## Truthy and Falsy — The Expanded Boolean Space

Python does not just evaluate `True` and `False` in boolean context. It evaluates **truthiness:**

| Value | Truthiness |
|-------|-----------|
| `0`, `0.0` | Falsy |
| `""`, `[]`, `{}`, `set()` | Falsy |
| `None` | Falsy |
| Everything else | Truthy |

```python
# This means you can write:
if username:            # True if username is not empty
    process(username)

if not errors:          # True if errors list is empty
    proceed()

items = get_items() or []   # Use empty list if get_items() returns None
```

This Pythonic pattern makes conditions expressive and concise.

## Control Flow as Philosophy

In Zen, the practitioner faces a fork in the road at every moment: *act or pause, speak or listen, push or yield.*

Control flow is that fork, formalized. Every `if` is a decision. Every `while` is a commitment. Every `for` is a complete traversal.

When you write control flow, you are encoding **judgment** into code. Not just mechanics — *wisdom about when to do what.*

The quality of your control flow reflects the quality of your thinking.

> *"The superior person does not advance without a clear path.*
> *Nor does they wait when the path is open."*
