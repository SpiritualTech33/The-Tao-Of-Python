# 4.6 Scope — The Art of the Limit

## Why Scope Exists

Scope defines **where a name is visible** in your code. It is not just a technical rule — it is an architectural principle.

Without scope, every variable in a program would be visible everywhere. Functions could accidentally modify variables from distant parts of the program. Debugging would be a nightmare. Reasoning about code would be impossible.

Scope creates **boundaries.** And boundaries create clarity.

---

## The LEGB Rule

Python searches for names in exactly this order:

1. **L**ocal — inside the current function
2. **E**nclosing — in any enclosing functions (for nested functions)
3. **G**lobal — at the module level
4. **B**uilt-in — Python's own built-ins (`len`, `print`, `type`, etc.)

```python
x = "global"   # Global scope

def outer():
    x = "enclosing"   # Enclosing scope

    def inner():
        x = "local"   # Local scope
        print(x)      # "local" — found in L

    inner()
    print(x)          # "enclosing" — found in E

outer()
print(x)              # "global" — found in G
```

---

## Local Scope

Variables defined inside a function exist only within it:

```python
def calculate():
    result = 42        # Local — only visible inside this function
    return result

# print(result)   # NameError — result doesn't exist here
```

This is the default and the most common. Local scope is the dojo wall — what happens inside stays inside.

---

## Global Scope

Module-level variables are accessible everywhere, but should rarely be *modified* from inside functions:

```python
TAX_RATE = 0.15    # Convention: UPPERCASE for module-level constants

def calculate_tax(amount: float) -> float:
    return amount * TAX_RATE   # Can READ global constant — fine

# Modifying global state — avoid when possible
counter = 0

def increment():
    global counter   # Required to modify a global from inside a function
    counter += 1

# Better design — make state explicit
def increment(counter: int) -> int:
    return counter + 1   # Pure. No global state.
```

---

## Closures — Enclosing Scope in Action

A closure is a function that remembers the variables from its enclosing scope, even after that scope has finished:

```python
def make_counter(start: int = 0):
    count = start   # Enclosing scope variable

    def counter() -> int:
        nonlocal count   # Tell Python: I want to modify the enclosing variable
        count += 1
        return count

    return counter   # Return the inner function

counter_a = make_counter(0)
counter_b = make_counter(10)

counter_a()   # 1
counter_a()   # 2
counter_b()   # 11  — independent from counter_a
```

`nonlocal` is the keyword for enclosing scope modification (as `global` is for global scope).

## The Philosophical Principle

The Tao Te Ching says:

> *"Knowing others is wisdom.*
> *Knowing yourself is enlightenment."*

Scope is self-knowledge for functions. A function with tight scope knows exactly what it owns, what it borrows, and what it does not touch.

**A function that reaches into global state to do its work is a function hiding its dependencies.** It looks simple but it is not. It has secret connections to the world outside.

The well-scoped function is honest: everything it needs comes through its parameters. Everything it produces comes through its return value. Nothing hidden. Nothing surprising.

---

## Practical Guidelines

1. **Prefer local scope** — create variables inside functions
2. **Use module-level constants** for values that are truly fixed (use UPPERCASE)
3. **Avoid `global`** except in rare cases — it creates hidden dependencies
4. **Pass state explicitly** through parameters — this makes dependencies visible
5. **Use closures** when you need stateful behavior without classes
