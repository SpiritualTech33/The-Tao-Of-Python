# 4.1 What Is a Function?

## The Primordial Question

Before syntax, ask why.

**Not:** "How do you write a function?"

**But:** "Why does the universe need functions?"

---

## Level 1: Technical Definition

A function is a **reusable block of code** that:
- Has a name
- Can receive data (parameters)
- Performs a specific task
- Can return a result

```python
def greet(name: str) -> str:
    return f"Hello, {name}!"

result = greet("Cosmos")   # "Hello, Cosmos!"
```

---

## Level 2: Functional Definition

Functions solve the problem of **repetition with variation.**

Without functions, every slightly different use of the same logic requires rewriting it:

```python
# Without functions — repetition
avg1 = 34 / 100
avg2 = 27 / 90
avg3 = 15 / 80
# Each line is the same pattern with different numbers

# With a function — one definition, infinite reuse
def batting_average(hits: int, at_bats: int) -> float:
    return hits / at_bats if at_bats > 0 else 0.0

avg1 = batting_average(34, 100)
avg2 = batting_average(27, 90)
avg3 = batting_average(15, 80)
```

---

## Level 3: Ontological Definition

Here is where it gets deep.

A function is how **abstraction enters your code.** When you name a function `calculate_batting_average`, you are lifting a pattern out of the concrete (specific numbers) into the abstract (the concept of average).

That is the same cognitive move that mathematics makes. And philosophy. And language itself.

When you write `batting_average(hits, at_bats)`, you are not specifying any particular hits or at_bats. You are describing a **relationship between concepts** — a truth that holds for all possible values.

That is abstraction. And abstraction is how minds grasp reality at a higher level.

---

## The DRY Principle

**Don't Repeat Yourself.** Every piece of knowledge should exist in exactly one place.

When logic is duplicated:
- A bug must be fixed in multiple places
- A change must be made in multiple places
- A reader must understand it multiple times

Functions enforce DRY. One definition. One place to fix. One place to understand.

---

## First-Class Functions

In Python, functions are **first-class citizens** — they can be:

```python
# Assigned to variables
double = lambda x: x * 2

# Passed as arguments
numbers = [1, 2, 3, 4, 5]
doubled = list(map(double, numbers))   # [2, 4, 6, 8, 10]

# Returned from other functions
def make_multiplier(factor: int):
    def multiply(x):
        return x * factor
    return multiply   # Returns a function

triple = make_multiplier(3)
print(triple(5))   # 15
```

This makes Python functions not just reusable blocks of code — but **objects** that can be composed, passed around, and built from other functions. This is the foundation of functional programming.
