# 1.6 String Operations

## Strings as the Interface

A string is the primary interface between your program and human consciousness. Every message a user reads, every log entry, every document your system processes — all strings. Master them.

## Indexing and Slicing

Strings are sequences. Each character has an index (starting at 0):

```python
word = "Python"
#       012345   (positive indices)
#      -654321   (negative indices)

print(word[0])     # "P"
print(word[-1])    # "n"  (last character)
print(word[2])     # "t"
```

**Slicing** extracts a portion: `string[start:stop:step]`

```python
word = "Python"

print(word[0:3])    # "Pyt"   (0 to 2, stop is exclusive)
print(word[2:])     # "thon"  (from 2 to end)
print(word[:4])     # "Pyth"  (from start to 3)
print(word[::2])    # "Pto"   (every 2nd character)
print(word[::-1])   # "nohtyP" (reversed)
```

## Essential Methods

```python
text = "  The Zen of Python  "

# Cleaning
text.strip()              # "The Zen of Python"
text.lstrip()             # "The Zen of Python  "
text.rstrip()             # "  The Zen of Python"

# Case
"cosmos".upper()          # "COSMOS"
"COSMOS".lower()          # "cosmos"
"cosmos de la cruz".title()    # "Cosmos De La Cruz"
"cosmos de la cruz".capitalize()  # "Cosmos de la cruz"

# Search
phrase = "I love Python"
phrase.find("Python")     # 7 (index), or -1 if not found
phrase.count("o")         # 2
"Python" in phrase        # True
phrase.startswith("I")    # True
phrase.endswith("n")      # True

# Modify (returns new string)
phrase.replace("Python", "Zen")   # "I love Zen"
```

## Split and Join — The Power Pair

```python
# split — break string into list
sentence = "the tao of programming"
words = sentence.split()          # ["the", "tao", "of", "programming"]

csv = "Cosmos,Miguel,Luis,Ana"
names = csv.split(",")            # ["Cosmos", "Miguel", "Luis", "Ana"]
parts = csv.split(",", 2)         # ["Cosmos", "Miguel", "Luis,Ana"] (max 2 splits)

# join — assemble list into string
words = ["chopping", "wood", "carrying", "water"]
" ".join(words)       # "chopping wood carrying water"
"-".join(words)       # "chopping-wood-carrying-water"
"".join(words)        # "choppingwoodcarryingwater"
```

`split()` and `join()` are inverses. Learn them as a pair — they appear constantly in real code.

## String Formatting — f-strings

Three styles exist. Use f-strings (Python 3.6+) by default:

```python
name = "Cosmos"
hits = 34
avg = 0.3421

# f-string — modern, clean, expressive
print(f"Player: {name}")
print(f"Hits: {hits}, Average: {avg:.3f}")   # .3f = 3 decimal places
print(f"Double hits: {hits * 2}")            # expressions work inside {}
print(f"Name length: {len(name)}")

# Alignment and padding
print(f"{'left':<10}")     # "left      " (left-aligned, 10 wide)
print(f"{'right':>10}")    # "     right" (right-aligned)
print(f"{'center':^10}")   # "  center  " (centered)
print(f"{3.14159:.2f}")    # "3.14"
print(f"{1000000:,}")      # "1,000,000" (comma separator)
```

## String as Immutable Sequence

Remember: strings cannot be changed after creation.

```python
name = "cosmos"
name[0] = "C"    # TypeError: 'str' object does not support item assignment
```

To "modify" a string, create a new one:
```python
name = "cosmos"
name = name[0].upper() + name[1:]   # "Cosmos"
# or
name = name.capitalize()            # "Cosmos"
```

This immutability is a feature, not a bug — it makes strings safe to pass around without unexpected modifications.
