# 3.3 String Formatting — Speaking with Precision

## Why Formatting Matters

Formatting is how code speaks to humans. A well-formatted output is clear, professional, and trustworthy. A poorly formatted one feels careless — even if the underlying logic is correct.

```python
# Unformatted — hard to read
print("Player: " + name + " Average: " + str(avg) + " Hits: " + str(hits))

# Formatted — clean and expressive
print(f"Player: {name} | Average: {avg:.3f} | Hits: {hits}")
```

## f-Strings — The Complete Guide

f-strings (formatted string literals) evaluate expressions inside `{}` at runtime:

```python
name = "Cosmos"
hits = 34
avg = 0.3421
salary = 1500000.50

# Basic substitution
f"Hello, {name}!"

# Expressions
f"Double hits: {hits * 2}"
f"Upper name: {name.upper()}"
f"Avg grade: {'elite' if avg >= 0.300 else 'developing'}"

# Calling functions
f"Name length: {len(name)}"
f"Initials: {name[0]}."
```

### Format Specifications

The power of f-strings is in the format spec after `:`:

```python
pi = 3.14159265358979

# Number formats
f"{pi:.2f}"         # "3.14"  — 2 decimal places
f"{pi:.5f}"         # "3.14159" — 5 decimal places
f"{pi:10.2f}"       # "      3.14" — 10 wide, 2 decimals
f"{pi:010.2f}"      # "0000003.14" — zero-padded

# Integer formats
n = 1000000
f"{n:,}"            # "1,000,000" — thousand separator
f"{n:_}"            # "1_000_000" — underscore separator
f"{n:b}"            # "11110100001001000000" — binary
f"{n:x}"            # "f4240" — hexadecimal
f"{n:o}"            # "3641100" — octal

# String alignment
f"{'left':<10}"     # "left      " — left-align in 10 chars
f"{'right':>10}"    # "     right" — right-align
f"{'center':^10}"   # "  center  " — center
f"{'fill':*^10}"    # "**fill****" — center with fill char

# Percentage
rate = 0.85
f"{rate:.1%}"       # "85.0%"
```

## Building Tables

f-strings make clean tabular output easy:

```python
players = [
    ("Cosmos", "pitcher",  0.342),
    ("Miguel", "catcher",  0.298),
    ("Luis",   "shortstop", 0.315),
]

# Header
print(f"{'Name':<10} {'Position':<12} {'AVG':>6}")
print("-" * 30)

# Rows
for name, position, avg in players:
    print(f"{name:<10} {position:<12} {avg:>6.3f}")

# Output:
# Name       Position       AVG
# ------------------------------
# Cosmos     pitcher      0.342
# Miguel     catcher      0.298
# Luis       shortstop    0.315
```

## Debugging with f-strings (Python 3.8+)

A lesser-known but powerful feature — the `=` specifier prints both the variable name and its value:

```python
x = 42
avg = 0.342
name = "Cosmos"

print(f"{x=}")       # "x=42"
print(f"{avg=:.3f}") # "avg=0.342"
print(f"{name=}")    # "name='Cosmos'"
```

This is faster than writing `print(f"x = {x}")` and identical in output. Useful for quick debugging without a debugger.
