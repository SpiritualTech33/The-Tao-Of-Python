# 4.5 *args and **kwargs — Flexibility Without Chaos

## The Problem of Unknown Quantity

Sometimes you don't know in advance how many arguments a function will receive. Python solves this elegantly with two special parameter forms.

---

## `*args` — Variable Positional Arguments

The `*` before `args` collects all extra positional arguments into a **tuple:**

```python
def sum_all(*numbers: float) -> float:
    """Sum any number of values."""
    return sum(numbers)

sum_all(1, 2, 3)           # 6.0
sum_all(10, 20, 30, 40)    # 100.0
sum_all(3.14)              # 3.14
sum_all()                  # 0.0

# Inside the function, numbers is a tuple
def show_args(*args):
    print(type(args))    # <class 'tuple'>
    print(args)
```

The name `args` is convention, not requirement. `*values`, `*items`, `*numbers` all work. The `*` is what matters.

---

## `**kwargs` — Variable Keyword Arguments

The `**` collects all extra keyword arguments into a **dictionary:**

```python
def create_profile(name: str, **attributes) -> dict:
    """Create a profile with any additional attributes."""
    profile = {"name": name}
    profile.update(attributes)
    return profile

create_profile("Cosmos", age=25, position="pitcher")
# {"name": "Cosmos", "age": 25, "position": "pitcher"}

create_profile("Cosmos")
# {"name": "Cosmos"}

# Inside the function, attributes is a dict
def show_kwargs(**kwargs):
    print(type(kwargs))    # <class 'dict'>
    for key, value in kwargs.items():
        print(f"  {key}: {value}")
```

---

## Full Signature Order

When combining all parameter types, order matters:

```python
def full_function(
    positional,           # Required positional
    *args,                # Extra positional (tuple)
    keyword_only,         # Keyword-only (after *)
    keyword_with_default="default",  # Keyword with default
    **kwargs              # Extra keyword (dict)
):
    pass
```

Python enforces this order. Violating it is a `SyntaxError`.

---

## Unpacking with * and **

The `*` and `**` also work **at call time** to unpack collections into arguments:

```python
def add(a, b, c):
    return a + b + c

# Unpack list into positional arguments
numbers = [1, 2, 3]
add(*numbers)    # Same as add(1, 2, 3)

# Unpack dict into keyword arguments
params = {"a": 1, "b": 2, "c": 3}
add(**params)    # Same as add(a=1, b=2, c=3)

# Merge two lists
list1 = [1, 2, 3]
list2 = [4, 5, 6]
combined = [*list1, *list2]   # [1, 2, 3, 4, 5, 6]

# Merge two dicts
dict1 = {"a": 1}
dict2 = {"b": 2}
merged = {**dict1, **dict2}   # {"a": 1, "b": 2}
```

---

## Practical Pattern — Passing Arguments Through

A common use case is passing arguments from one function to another:

```python
def log_and_call(func, *args, **kwargs):
    """Call func with any arguments, logging before and after."""
    print(f"Calling {func.__name__}...")
    result = func(*args, **kwargs)
    print(f"Done. Result: {result}")
    return result

log_and_call(sum_all, 1, 2, 3, 4, 5)
# Calling sum_all...
# Done. Result: 15
```

This pattern — wrapping a function call — is the foundation of decorators, middleware, and many design patterns.
