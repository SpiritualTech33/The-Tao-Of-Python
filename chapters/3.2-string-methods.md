# 3.2 String Methods — The Complete Toolkit

## The Standard Toolkit

These are the methods you will use every day. Not memorized — internalized.

### Transformation

```python
s = "the zen of python"

s.upper()          # "THE ZEN OF PYTHON"
s.lower()          # "the zen of python"
s.title()          # "The Zen Of Python"
s.capitalize()     # "The zen of python"
s.swapcase()       # "THE ZEN OF PYTHON" → "the zen of python"
```

### Cleaning

```python
raw = "   hello world   "

raw.strip()         # "hello world"   — both sides
raw.lstrip()        # "hello world   " — left only
raw.rstrip()        # "   hello world"  — right only
raw.strip("   ")    # same as strip() with spaces
"..hello..".strip(".")   # "hello"
```

### Search and Replace

```python
text = "I love Python. Python is great."

text.find("Python")          # 7
text.rfind("Python")         # 15  (rightmost)
text.count("Python")         # 2
text.replace("Python", "Zen")  # "I love Zen. Zen is great."
text.replace("Python", "Zen", 1)  # Replace only first occurrence
```

### Split and Join

```python
# Split by whitespace (default)
"  the   tao  ".split()       # ["the", "tao"]

# Split by delimiter
"a,b,c,d".split(",")          # ["a", "b", "c", "d"]
"a,b,c,d".split(",", 2)       # ["a", "b", "c,d"]

# Split into at most N+1 pieces
"one::two::three".split("::", 1)  # ["one", "two::three"]

# Join
"-".join(["a", "b", "c"])     # "a-b-c"
"\n".join(["line1", "line2"]) # "line1\nline2"
```

### Padding and Alignment

```python
"42".zfill(5)           # "00042"   — zero-pad on left
"zen".ljust(10)         # "zen       " — left-align, pad right
"zen".rjust(10)         # "       zen" — right-align, pad left
"zen".center(10)        # "   zen    " — center, pad both
"zen".center(10, "-")   # "---zen----" — center with fill char
```

## The `str.format()` Method

Before f-strings, `format()` was standard. You will encounter it in older code:

```python
name = "Cosmos"
avg = 0.342

"{} has an average of {:.3f}".format(name, avg)
# "Cosmos has an average of 0.342"

"{name} is a {role}".format(name="Cosmos", role="pitcher")
# "Cosmos is a pitcher"

"{0} and {0} again".format("zen")
# "zen and zen again"
```

Prefer f-strings for new code. Know `format()` to read old code.

## Building Strings Efficiently

When concatenating many strings, avoid using `+` in a loop:

```python
# Bad — creates a new string object on each iteration
result = ""
for word in words:
    result += word + " "   # N intermediate string objects

# Good — build a list, join once
parts = []
for word in words:
    parts.append(word)
result = " ".join(parts)   # One final string

# Best — list comprehension + join
result = " ".join(word for word in words)
```

This matters when building large strings. For a few concatenations, `+` is fine.
