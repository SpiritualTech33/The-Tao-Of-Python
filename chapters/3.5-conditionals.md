# 3.5 Conditionals

## The if Statement — The Fork in the Road

```python
score = 85

if score >= 90:
    print("Excellent")
elif score >= 70:
    print("Good")
elif score >= 50:
    print("Passing")
else:
    print("Keep practicing")
```

Rules:
- `if` is required. `elif` and `else` are optional.
- `elif` can appear any number of times.
- `else` is the fallback — runs if nothing above matched.
- Indentation (4 spaces) defines the block. This is Python's syntax.

## Nested Conditionals

Conditionals can be nested — but depth kills readability. **Prefer early returns and flat structure.**

```python
# Deeply nested — hard to follow
def classify(player):
    if player is not None:
        if player["active"]:
            if player["avg"] >= 0.300:
                return "elite"
            else:
                return "solid"
        else:
            return "inactive"
    return "unknown"

# Flat — linear, readable
def classify(player):
    if player is None:
        return "unknown"
    if not player["active"]:
        return "inactive"
    if player["avg"] >= 0.300:
        return "elite"
    return "solid"
```

Both functions do the same thing. The second is profoundly easier to read and maintain.

## The Ternary Expression

A one-line conditional for simple assignments:

```python
# Long form
if age >= 18:
    status = "adult"
else:
    status = "minor"

# Ternary — same result
status = "adult" if age >= 18 else "minor"

# Nested ternary — acceptable if very clear, avoid if complex
grade = "A" if score >= 90 else "B" if score >= 80 else "C"
```

Use ternary when it improves clarity. Avoid it when it makes the line hard to parse.

## Pattern: Guard Clauses

A guard clause is an early return that eliminates an invalid case before the main logic:

```python
def calculate_average(hits: int, at_bats: int) -> float:
    # Guards — handle invalid cases immediately
    if at_bats == 0:
        return 0.0
    if hits < 0 or at_bats < 0:
        raise ValueError("Stats cannot be negative")
    if hits > at_bats:
        raise ValueError("Hits cannot exceed at_bats")

    # Main logic — only reached if all guards passed
    return hits / at_bats
```

Guards make the happy path immediately visible. The main logic is not buried in `else` branches.

## match Statement (Python 3.10+)

Python's structural pattern matching — for complex conditional logic based on structure:

```python
def describe_position(position: str) -> str:
    match position:
        case "pitcher" | "catcher":
            return "battery"
        case "shortstop" | "second" | "first" | "third":
            return "infield"
        case "left" | "center" | "right":
            return "outfield"
        case _:
            return "unknown position"
```

`match` is more powerful than `if/elif` chains for pattern-based logic. The `_` is the wildcard — matches anything.

## Common Mistakes

```python
# Mistake 1 — assignment in condition (use == not =)
if x = 5:    # SyntaxError
if x == 5:   # Correct

# Mistake 2 — comparing to True/False unnecessarily
if is_active == True:   # Verbose
if is_active:           # Pythonic

# Mistake 3 — comparing to None with ==
if result == None:   # Works but wrong
if result is None:   # Correct — None is a singleton
```
