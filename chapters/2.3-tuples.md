# 2.3 Tuples

## Why Immutability Exists

A tuple is like a list — but once created, it **cannot change.**

Why would you ever want that? Because some things *should not change.* The coordinates of a point. The RGB values of a color. The days of the week. The date of a game.

Immutability is not a restriction — it is a **declaration of intent.** When you use a tuple, you are saying to everyone who reads your code: *"This data is fixed by design. Do not modify it."*

```python
# List — can change
batting_order = ["Cosmos", "Miguel", "Luis"]
batting_order[0] = "Ana"   # Valid — lists are mutable

# Tuple — cannot change
home_plate = (0.0, 0.0)
# home_plate[0] = 1.0    → TypeError: 'tuple' object does not support item assignment
```

---

## Creating Tuples

```python
# Standard syntax
coordinates = (19.4326, -99.1332)   # Mexico City
rgb_red = (255, 0, 0)
dimensions = (1920, 1080)

# Tuple without parentheses (the comma creates the tuple)
point = 10, 25   # Valid tuple: (10, 25)

# Single-element tuple (requires trailing comma!)
singleton = (42,)   # Tuple with one element
not_a_tuple = (42)  # This is just the integer 42
```

## Tuple Unpacking

One of Python's most elegant features. Assign multiple variables from a tuple in one line:

```python
coordinates = (19.4326, -99.1332)
latitude, longitude = coordinates

print(latitude)    # 19.4326
print(longitude)   # -99.1332

# Swap two variables without a temporary variable
a, b = 1, 2
a, b = b, a
print(a, b)   # 2 1

# Unpack with * to capture the rest
first, *middle, last = (1, 2, 3, 4, 5)
print(first)    # 1
print(middle)   # [2, 3, 4]
print(last)     # 5
```

## When to Use Tuple vs List

**Use a tuple** when:
- The data represents a single composite unit (a coordinate, an RGB color, a point in time)
- You want to signal "this should not change"
- You need to use the collection as a dictionary key (lists can't be dict keys — tuples can)
- Returning multiple values from a function

**Use a list** when:
- The collection will grow, shrink, or be reordered
- You need to sort, append, or remove elements

```python
# Tuple — fixed composite unit
player_record = ("Cosmos", 0.342, 34, 100)   # name, avg, hits, at_bats

# List — changing sequence
active_players = ["Cosmos", "Miguel", "Luis"]
active_players.append("Ana")   # team grows

# Function returning multiple values — Python uses tuple
def get_stats(hits, at_bats):
    average = hits / at_bats if at_bats else 0.0
    obp = (hits * 1.1) / at_bats if at_bats else 0.0
    return average, obp   # Returns a tuple

avg, obp = get_stats(34, 100)   # Unpacked immediately
```

---

## Tuples Are Faster

Tuples are slightly faster than lists because Python can optimize immutable structures. For large datasets processed many times, this matters. For small programs, it rarely does.

Use the one that best expresses your intent. Performance is secondary to clarity.
